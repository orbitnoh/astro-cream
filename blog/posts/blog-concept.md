## 이런 블로그입니다

단순히 마크다운을 렌더링하는 테마가 아니라, 글의 작성부터 관리까지 흐름을 지원합니다.  
날짜순 블로그는 글을 쌓기엔 편하지만, 글이 흩어져 있어 체계가 부족합니다.

이 템플릿은 날짜순 뿐만 아니라 글을 주제별로 묶고 순서를 직접 정할 수 있도록 설계했습니다.  
글쓰기와 정리는 분리되어 있어, 관리의 부담을 낮췄습니다.

## 어울리는 용도

- 공부한 내용을 순서 있게 정리하는 블로그
- 배운 내용을 개인 지식 저장소처럼 관리하는 블로그
- 시리즈 글과 단발성 메모가 함께 쌓이는 블로그
- 태그나 날짜순 목록만으로는 정리가 부족한 블로그

## 핵심 개념

### 하나의 목차

모든 글의 링크는 `blog/archive.md`에서 한 번에 관리합니다.  
주제마다 별도의 목차 파일을 만들지 않아도 되고, 글의 위치를 바꾸기 위해 여러 파일을 오갈 필요도 없습니다.

목차는 단순한 글 목록의 나열이 아니라, 블로그 전체의 지도를 만드는 공간입니다.

### 글 작성과 정리를 분리하는 Inbox

확장 프로그램을 통해 새 글을 만들면 링크가 `blog/archive.md`의 `Inbox` 영역에 자동으로 추가됩니다.  
글을 작성하는 순간은 분류를 고민하기 좋은 순간이 아닙니다.

작성한 글은 일단 `Inbox`에 모아두고, 나중에 글의 역할이 분명해졌을 때 알맞은 주제로 옮깁니다.  
이렇게 글쓰기와 정리를 분리하면, 미루는 것이 게으름이 아닌 전략적 선택이 되며 점진적으로 목차를 다듬어 갈 수 있습니다.

## 사용 흐름

이 템플릿은 [Astro Cream CMS Tools]() VS Code 확장 프로그램과 함께 사용하는 것을 권장합니다.  
글은 하나지만 frontmatter를 사용하는 곳은 글 마크다운과 `blog/archive.md` 두 곳이기 때문에, 확장 프로그램으로 함께 관리하는 흐름을 전제로 합니다.

확장 프로그램으로 새 글을 만들면 글 마크다운이 생성되고, 동시에 `blog/archive.md`의 `Inbox` 영역에 링크가 자동으로 추가됩니다.  
이후 제목, 공개 여부, slug 같은 frontmatter를 수정할 때도 확장 프로그램을 사용하면 글 파일과 `blog/archive.md`의 값이 함께 갱신됩니다.

사용자는 새 글의 위치를 처음부터 고민하지 않아도 됩니다.  
일단 `Inbox`에 쌓아두고, 글의 역할이 분명해졌을 때 `blog/archive.md`에 주제 헤딩을 만든 뒤 링크를 원하는 위치로 옮기면 됩니다.
